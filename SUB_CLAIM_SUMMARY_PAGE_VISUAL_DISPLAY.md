## How Data Flows to the Page

### Step 1: User Clicks Claim in Dashboard
```
Dashboard
    ?
Click "CLM202512211001"
    ?
Navigate to /claim/CLM202512211001
```

### Step 2: Page Loads
```
ClaimDetail.razor Component Initializes
    ?
OnInitializedAsync() executes
    ?
ClaimService.GetClaimAsync("CLM202512211001") called
    ?
MockClaimService returns Claim object
    ?
PopulateSubClaims() generates 4 sample sub-claims
    ?
Claim.SubClaims = [BI, PIP, PD, UM]
```

### Step 3: Page Renders
```
Header Section renders
    ? Claim Summary shows (CLM202512211001, CAF001711, etc.)
    ?
Navigation Tabs render
    ? "Sub-Claims Summary" tab is visible
    ?
Tab Content renders (with active tab on Loss Details)
    ?
User clicks "Sub-Claims Summary" tab
    ?
SubClaimsSummarySection.razor renders
    ?
Inner tabs appear (Sub-Claims Grid, Working Sub-Claims)
    ?
SubClaimsSummaryGrid.razor renders with 4 sub-claims
    ?
Grid displays with all data populated
```

### Step 4: User Interacts
```
User can:
    ? View all sub-claim data
    ? Select rows with checkboxes
    ? Trigger action menu
    ? Click status badges
    ? Navigate pagination
    ? Switch between tabs
```
