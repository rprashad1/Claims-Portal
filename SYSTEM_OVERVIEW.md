# Claims Portal - System Overview

## Executive Summary

The Claims Portal is a modern, web-based insurance claims management system built with .NET 10 and Blazor Server. It provides a complete workflow for managing First Notice of Loss (FNOL) claims from initial intake through resolution.

## User Workflows

### Creating a New Claim
```
1. User clicks "FNOL" in navigation
2. User clicks "New FNOL" button
3. System generates FNOL number (FNOL{date}{seq})
4. User completes Step 1: Loss Details
   - Enters date, time, location
   - Adds witnesses (optional)
   - Adds authorities (optional)
5. User completes Step 2: Policy & Insured
   - Searches for policy
   - System displays policy/insured info
   - Selects or enters vehicle
6. User completes Step 3: Driver & Injury
   - Enters driver information
   - Adds injury details (if applicable)
   - Adds attorney (if applicable)
   - Adds passengers (if applicable)
7. User completes Step 4: Third Parties
   - Adds other vehicles
   - Adds pedestrians/bicyclists
   - Adds property damage
8. User completes Step 5: Review & Save
   - Reviews all information
   - Creates sub-claims/features
   - Submits claim
9. System generates Claim Number
10. Success modal displayed
```

### Viewing a Claim
```
1. User clicks "FNOL" in navigation
2. User enters search criteria
3. User clicks "Search"
4. System displays matching claims
5. User clicks "View" on desired claim
6. System loads claim details
7. User navigates between tabs:
   - Loss Details
   - Insured & Vehicle
   - Driver & Passengers
   - Third Parties
   - Sub-Claims Summary
   - Financials
   - Claims Log
   - Communication
```

## Performance Considerations

- **Pagination**: All grids support pagination
- **Lazy Loading**: Data loaded on demand
- **Caching**: Lookup codes cached where appropriate
- **Indexing**: Database indexes on frequently queried columns
- **Async Operations**: Non-blocking I/O operations
